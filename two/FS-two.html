Firestorm Two

Adapt your existing DataPerfect database to run on the Internet.

Overview

A Firestorm Two web application runs on a webserver. It is based on a DataPerfect database. The users run a browser such as Internet Explorer and fill out forms to provide information to or request information from the FS2 web application.

A Firestorm Two application is a series of webpages. Some of the webpages are static HTML. The rest of the webpages are generated by DataPerfect reports. Each page has an HTML form with named fields. When the user clicks on a Sumbit button, the fields are sent by the browser to the server where the Firestorm Two application is running. The fields are received by the webserver, which calls the FS2 server, which creates a temporary t-log file and calls the DataPerfect report runner.

Conversion Steps

  1. Add fields to the user ID panel so that it can be used for session tracking.
  2. Add a panel to hold parameters from browser requests.
  3. Write HTML for initial webpages.
  4. Write reports to generate follow-on webpages.
  5. Write a configuration file specifying panel numbers and parameter names.
